Leetcode 49（Group Anagrams）问题的最佳做法通常是 **使用排序法** 或 **字符计数法** 来分组字母重排的字符串。这两种方法都能高效、准确地解决问题。

### 1. **排序法**（最常见的方法）

通过对每个字符串进行排序，所有字母重排的字符串都会变成相同的字符串，从而可以用排序后的字符串作为哈希表的键。

#### 代码实现：

```cpp
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        unordered_map<string, vector<string>> umap;
        
        for (const string& s : strs) {
            string sorted_s = s;  // 将字符串复制一份用于排序
            sort(sorted_s.begin(), sorted_s.end());  // 对字符串排序
            umap[sorted_s].push_back(s);  // 将原始字符串添加到排序后的键对应的组中
        }
        
        vector<vector<string>> res;
        for (const auto& [key, vec] : umap) {
            res.push_back(vec);  // 把每个分组添加到结果中
        }
        
        return res;
    }
};
```

#### 解释：

1. **排序**：对每个字符串进行排序，得到一个标准化的字符串。如果两个字符串是字母重排，它们排序后的字符串是相同的。
2. **哈希表**：使用 `unordered_map`，键是排序后的字符串，值是所有字母重排的字符串组成的列表。这样所有字母重排的字符串会被放在同一个组里。
3. **结果构建**：最后通过遍历哈希表，构造出包含所有字母重排分组的二维列表 `res`。

#### 时间复杂度：

* 排序每个字符串的时间复杂度是 O(K log K)，其中 K 是字符串的长度。
* 假设有 N 个字符串，总的时间复杂度是 O(N * K log K)。

#### 空间复杂度：

* 哈希表和结果数组占用的空间是 O(N * K)，其中 N 是字符串的数量，K 是字符串的平均长度。

---

### 2. **字符计数法**（另一种有效方法）

如果你不想进行排序，可以使用 **字符计数法**，即记录每个字符串中字母的出现频次，然后用这些频次作为哈希表的键。这样，字母重排的字符串会有相同的字符频率，从而能够分到同一组。

#### 代码实现：

```cpp
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        unordered_map<string, vector<string>> umap;
        
        for (const string& s : strs) {
            vector<int> count(26, 0);  // 用一个长度为26的数组来记录字母的频次
            for (char c : s) {
                count[c - 'a']++;  // 更新字符频次
            }
            
            string key = "";
            for (int i : count) {
                key += to_string(i) + "#";  // 将字符频次拼接成字符串作为哈希表的键
            }
            
            umap[key].push_back(s);  // 将原始字符串添加到频次对应的组中
        }
        
        vector<vector<string>> res;
        for (const auto& [key, vec] : umap) {
            res.push_back(vec);  // 把每个分组添加到结果中
        }
        
        return res;
    }
};
```

#### 解释：

1. **字符计数**：对于每个字符串，使用一个长度为 26 的数组记录每个字母出现的次数。字符 'a' 对应数组的第 0 位，'b' 对应第 1 位，以此类推。
2. **生成键**：将字符频次数组转化为一个字符串，用它作为哈希表的键。这个字符串是唯一的，可以唯一标识一个字母重排的组。
3. **哈希表**：用哈希表将字符频次相同的字符串分为一组。
4. **结果构建**：最后遍历哈希表，构建出所有分组的结果。

#### 时间复杂度：

* 对每个字符串，计算字符频次的时间复杂度是 O(K)，其中 K 是字符串的长度。
* 假设有 N 个字符串，总的时间复杂度是 O(N * K)。

#### 空间复杂度：

* 哈希表和字符频次数组占用的空间是 O(N * K)，其中 N 是字符串的数量，K 是字符串的平均长度。

---

### 选择排序法还是字符计数法？

* **排序法**：相对简单，且容易理解，适用于字母集较小的情况（比如字母表中的字母）。在排序过程中，字符串的长度较短时比较迅速。
* **字符计数法**：适合在字母表非常大的情况下（例如对于 Unicode 字符）或者要求不进行排序时。字符计数法需要额外的空间来存储字符频次数组。

### 总结：

对于 **Leetcode 49** 问题，**排序法** 和 **字符计数法** 都是最佳的解决方法。两者时间复杂度接近，但排序法更常用且简单。如果对性能要求极高，可以选择字符计数法。
