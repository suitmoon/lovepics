你这段代码用的是**滑动窗口/双指针**思路：`sum>k` 就收缩左端 `front++`。
但 **“和为 k 的子数组” 这个题（LeetCode 560）里，nums 可以有负数/0**，一旦有负数，`sum>k` 并不意味着“再扩大只会更大”，所以滑动窗口的单调性条件不成立，逻辑就会漏解/错解。

## 为什么会错（核心原因）

滑动窗口成立的前提通常是：数组元素 **全是非负数**（或至少保证窗口扩大时 sum 单调不减）。
而这里可能出现负数：

* 你看到 `sum > k` 就把左边删掉
* 但后面如果来了一个负数，`sum` 可能又降回 `k`
* 你提前收缩窗口，答案就被你“删没了”

## 一个反例（你的代码会漏）

`nums = [1, -1, 1] , k = 1`

正确答案：有 3 个子数组和为 1：

* `[1]` (0..0)
* `[1, -1, 1]` (0..2)
* `[1]` (2..2)

你的代码过程大概是：

* i=0: sum=1 ==k -> count=1，随后你还做了 `sum-=nums[front++]`，sum 变 0
* i=1: sum+=-1 -> -1
* i=2: sum+=1 -> 0 （永远回不到你想要的那种窗口状态）
  最终只数到 1

这里除了“负数破坏单调性”，你还有个额外问题：

### 你在 `sum==k` 时强行 `sum-=nums[front++]`

这会把“可能继续扩展得到更多解”的窗口提前破坏掉（即使全是非负数，也会影响统计多解的方式）。

---

## 正确做法（前缀和 + 哈希计数）

这题标准解法是：统计前缀和 `preSum`，如果存在某个之前的前缀和等于 `preSum - k`，就说明中间那段子数组和为 k。

C++ 正确代码：

```cpp
class Solution {
public:
    int subarraySum(vector<int>& nums, int k) {
        unordered_map<int,int> cnt;
        cnt[0] = 1;              // 前缀和为0出现1次（空前缀）
        int pre = 0, ans = 0;
        for (int x : nums) {
            pre += x;
            if (cnt.count(pre - k)) ans += cnt[pre - k];
            cnt[pre]++;
        }
        return ans;
    }
};
```

时间 O(n)，对负数/0 都正确。

## 什么时候你的滑动窗口思路才对？

如果题目保证 `nums[i] >= 0`（全非负），才可以用双指针去做“和为 k”的计数/查找（但计数也要更小心处理重复窗口）。
