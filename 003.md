好，我们就**沿着你现在的想法**来走：你在做的是“每次把新数字加进来，更新所有以当前位置结尾的子数组和”。这个方向其实非常对！只是你现在写法是“把所有和都列出来”，会变成 O(n²)。

我先把你写的过程**翻译成标准结构**，然后告诉你怎么从这一步“升级”到题解里的 O(n)。

## 1）你现在在做什么（你列的那些数到底是什么）

nums = `[1,2,3]`

你列：

* 看到 1：
  `1`
  （表示：以 1 结尾的子数组只有 `[1]`，和是 1）

* 再看到 2：
  `1,2,3`
  这其实是：

  * `[1]` 的和 1（旧的）
  * `[2]` 的和 2（新开一段）
  * `[1,2]` 的和 3（把旧的 1 加上 2）

* 再看到 3：
  `1,3,5,3`
  这一步你其实在生成“所有子数组的和”了：

  * 旧的 `[1]` 还是 1
  * 旧的 `[2]` 加 3 变成 `[2,3]`：2+3=5
  * 旧的 `[1,2]` 加 3 变成 `[1,2,3]`：3+3=6（你写里没出现，说明你列表里有点混了）
  * 新开 `[3]`：3
    所以正确的“以 3 结尾的子数组和”应该是：
    `[3]=3, [2,3]=5, [1,2,3]=6`

**结论：你的思路本质是：**

> 维护一个列表 `endSums`，它保存“所有以当前下标 r 结尾的子数组和”。

每来一个新数 x：

* 新的 endSums = （旧的每个和都 +x）再加上（新开一段 x）

这就是典型 O(n²)。

## 2）你想要的是：不用把 endSums 全列出来，也能数出“等于 k 的个数”

关键问题：你列 endSums 的目的是什么？
是为了检查里面有多少个值等于 k，对吧？

比如到 r 时：

* 你要知道：以 r 结尾的子数组里，有多少个和是 k

如果我们能**不展开列表**，直接算出这个数量，就赢了。

## 3）把你“列表更新”的过程，换成“前缀和差”的表达（从你熟悉的出发）

你维护 endSums 时，其实每个子数组都形如：

`nums[l..r]`

它的和等于：

`pre[r] - pre[l-1]`

这里 `pre[r]` 是从 0 加到 r 的总和。

**现在想象你站在 r：**
`pre[r]` 已经确定了（因为你一路加过来）。

你要找多少个 l 使得：

`pre[r] - pre[l-1] = k`

移一下：

`pre[l-1] = pre[r] - k`

这句很重要，我用你的“列表视角”再说一遍：

> 你 endSums 里每一个值，其实都对应某个 l。
> 与其把每个 l 的子数组和都算出来，不如反过来问：
> “有哪些 l 能让和等于 k？”
> 等价于问：以前出现过多少次前缀和等于 `pre[r]-k`。

所以我们只要知道：
**“以前的前缀和取值分布（出现次数）”**
就能立刻知道以 r 结尾有多少个答案。

## 4）用你这个例子 `[1,2,3]`，我按“你能跟得上的步子”跑一遍

假设 k = 3（因为这个例子最直观）

我们维护两个东西：

* `pre`：当前前缀和
* `cnt`：某个前缀和出现了几次

初始化：

* `pre = 0`
* `cnt[0] = 1`（空前缀出现一次）
* `ans = 0`

### 走到 1

* `pre = 1`
* 想找：`pre - k = 1 - 3 = -2`
* `cnt[-2]` 没有 → 以 1 结尾没有和为 3 的子数组
* 把当前 `pre=1` 记入：`cnt[1]++`

此时：

* cnt: {0:1, 1:1}
* ans: 0

### 走到 2

* `pre = 3`
* 想找：`pre - k = 3 - 3 = 0`
* `cnt[0] = 1` → 说明以前有 1 次前缀和为 0

  * 那次就是“空前缀”，对应 l=0
  * 子数组就是 `[1,2]`，和为 3
* 所以 `ans += 1`
* 记入 `cnt[3]++`

此时：

* cnt: {0:1, 1:1, 3:1}
* ans: 1

### 走到 3

* `pre = 6`
* 想找：`pre - k = 6 - 3 = 3`
* `cnt[3] = 1` → 以前前缀和为 3 出现过 1 次（在走到 2 的时候）

  * 那次对应 l=2（因为 pre[1]=3）
  * 子数组就是 `[3]`，和为 3
* `ans += 1`
* 记入 `cnt[6]++`

最终 ans = 2
对应子数组：`[1,2]` 和 `[3]`

## 5）把它和你“endSums 列表”对照一下（你会一下明白）

当 r=1（元素2），你列的“以 r 结尾”的和是：

* `[2] = 2`
* `[1,2] = 3`  ✅ 有一个等于 3

我们的算法做的是：

* 不列 `[2]` 和 `[1,2]`
* 直接通过 `pre=3` 去问“以前有没有前缀和=0”
* 有 1 个 → 立刻得到 1 个答案

当 r=2（元素3），以 r 结尾的和是：

* `[3]=3` ✅
* `[2,3]=5`
* `[1,2,3]=6`

我们的算法做的是：

* `pre=6`
* 问“以前有没有前缀和=3”
* 有 1 个 → 立刻得到 `[3]` 这一段

所以它不是“推进很玄学”，而是：

> 你原本在维护一堆“以 r 结尾的所有和”，
> 它改成维护“所有历史前缀和出现次数”，
> 用一次查询就得到“以 r 结尾有多少个和为 k”。

---

## 6）最后把你的困惑点直接点破

你觉得你的方式是穷举，因为你在做：

* “所有以 r 结尾的子数组”都算一遍

而题解在做：

* “所有以 r 结尾、和为 k 的子数组”有多少个
  **直接用次数算出来**

它不去算那些不等于 k 的。
